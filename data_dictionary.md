# Mutual Fund Analytics Database - Data Dictionary

This document details the SQLite star schema used to model the Mutual Fund Analytics dataset. The database consists of 2 Dimension tables and 4 Fact tables.

---

## 1. Dimension Tables

### `dim_fund`
Master list of mutual fund schemes. Populated from `data/raw/01_fund_master.csv`.

| Column Name | SQLite Data Type | Constraint | Business Definition | Source Field |
| :--- | :--- | :--- | :--- | :--- |
| `fund_id` | `INTEGER` | Primary Key, Auto-increment | Unique identifier generated by the database | Generated |
| `amfi_code` | `TEXT` | Unique | Association of Mutual Funds in India code for the scheme | `amfi_code` |
| `scheme_name` | `TEXT` | - | Full name of the mutual fund scheme | `scheme_name` |
| `fund_house` | `TEXT` | - | The Asset Management Company (AMC) managing the fund | `fund_house` |
| `category` | `TEXT` | - | Broad asset class category (Equity, Debt, Hybrid, etc.) | `category` |
| `sub_category` | `TEXT` | - | Specific fund type (Large Cap, Small Cap, Gilt, Liquid, etc.) | `sub_category` |
| `plan` | `TEXT` | - | Type of distribution plan (Regular or Direct) | `plan` |
| `launch_date` | `TEXT` | - | Launch date of the fund (YYYY-MM-DD) | `launch_date` |
| `benchmark` | `TEXT` | - | Index against which fund performance is measured | `benchmark` |
| `expense_ratio_pct` | `REAL` | - | Operating fee as a percentage of assets | `expense_ratio_pct` |
| `exit_load_pct` | `REAL` | - | Fee charged to investors on redemption | `exit_load_pct` |
| `min_sip_amount` | `REAL` | - | Minimum investment for Systematic Investment Plans (INR) | `min_sip_amount` |
| `min_lumpsum_amount`| `REAL` | - | Minimum initial lump sum investment (INR) | `min_lumpsum_amount` |
| `fund_manager` | `TEXT` | - | Primary portfolio manager of the fund | `fund_manager` |
| `risk_grade` | `TEXT` | - | Risk rating classification (Low, Moderate, High, Very High) | `risk_category` |
| `sebi_category_code`| `TEXT` | - | Standardized category code specified by SEBI | `sebi_category_code` |

### `dim_date`
Calendar dimensions for standard reporting. Populated from dates found across NAV, transactions, and AUM datasets.

| Column Name | SQLite Data Type | Constraint | Business Definition | Source |
| :--- | :--- | :--- | :--- | :--- |
| `date_id` | `INTEGER` | Primary Key, Auto-increment | Unique identifier generated by the database | Generated |
| `date` | `TEXT` | Unique | Date representation in YYYY-MM-DD | Unified Date |
| `year` | `INTEGER` | - | Calendar year (e.g. 2024) | Generated |
| `month` | `INTEGER` | - | Calendar month number (1-12) | Generated |
| `day` | `INTEGER` | - | Calendar day number (1-31) | Generated |
| `quarter` | `INTEGER` | - | Calendar quarter number (1-4) | Generated |

---

## 2. Fact Tables

### `fact_nav`
Historical Net Asset Value (NAV) records. Populated from `data/raw/02_nav_history.csv` (with weekend/holiday gaps forward-filled).

| Column Name | SQLite Data Type | Constraint | Business Definition | Source Field |
| :--- | :--- | :--- | :--- | :--- |
| `nav_id` | `INTEGER` | Primary Key, Auto-increment | Unique identifier generated by the database | Generated |
| `fund_id` | `INTEGER` | Foreign Key (`dim_fund.fund_id`) | Mapped fund identifier | Mapped from `amfi_code` |
| `date_id` | `INTEGER` | Foreign Key (`dim_date.date_id`) | Mapped date identifier | Mapped from `date` |
| `nav` | `REAL` | - | Net Asset Value per unit on this date (INR) | `nav` |

### `fact_transactions`
Investor transaction ledger. Populated from `data/raw/08_investor_transactions.csv`.

| Column Name | SQLite Data Type | Constraint | Business Definition | Source Field |
| :--- | :--- | :--- | :--- | :--- |
| `tx_id` | `INTEGER` | Primary Key, Auto-increment | Unique identifier generated by the database | Generated |
| `investor_id` | `TEXT` | - | Unique identifier of the investor | `investor_id` |
| `fund_id` | `INTEGER` | Foreign Key (`dim_fund.fund_id`) | Mapped fund identifier | Mapped from `amfi_code` |
| `date_id` | `INTEGER` | Foreign Key (`dim_date.date_id`) | Mapped date identifier | Mapped from `transaction_date` |
| `transaction_type`| `TEXT` | - | Standardized type (SIP, Lumpsum, Redemption) | `transaction_type` |
| `amount_inr` | `REAL` | - | Transaction value (INR) | `amount_inr` |
| `kyc_status` | `TEXT` | - | Standardized KYC state (Verified, Pending, Rejected, Unknown) | `kyc_status` |
| `state` | `TEXT` | - | Indian state of residence of the investor | `state` |
| `city` | `TEXT` | - | City of residence of the investor | `city` |
| `city_tier` | `TEXT` | - | City category code (e.g. T30, B30) | `city_tier` |
| `age_group` | `TEXT` | - | Age bracket of the investor | `age_group` |
| `gender` | `TEXT` | - | Gender of the investor (Male, Female, Other) | `gender` |
| `annual_income_lakh`| `REAL` | - | Annual income of the investor in lakhs | `annual_income_lakh` |
| `payment_mode` | `TEXT` | - | Mode of payment used (Net Banking, UPI, Cheque, Mandate) | `payment_mode` |

### `fact_performance`
Scheme-level return and risk attributes. Populated from `data/raw/07_scheme_performance.csv`.

| Column Name | SQLite Data Type | Constraint | Business Definition | Source Field |
| :--- | :--- | :--- | :--- | :--- |
| `perf_id` | `INTEGER` | Primary Key, Auto-increment | Unique identifier generated by the database | Generated |
| `fund_id` | `INTEGER` | Foreign Key (`dim_fund.fund_id`) | Mapped fund identifier | Mapped from `amfi_code` |
| `return_1yr_pct` | `REAL` | - | 1-year annualized return percentage | `return_1yr_pct` |
| `return_3yr_pct` | `REAL` | - | 3-year annualized return percentage | `return_3yr_pct` |
| `return_5yr_pct` | `REAL` | - | 5-year annualized return percentage | `return_5yr_pct` |
| `benchmark_3yr_pct`| `REAL` | - | Benchmark return over 3 years | `benchmark_3yr_pct` |
| `alpha` | `REAL` | - | Excess return of the fund relative to the benchmark index | `alpha` |
| `beta` | `REAL` | - | Volatility measure relative to the market index | `beta` |
| `sharpe_ratio` | `REAL` | - | Risk-adjusted return measure | `sharpe_ratio` |
| `sortino_ratio` | `REAL` | - | Volatility adjusted return focusing on downside risk | `sortino_ratio` |
| `std_dev_ann_pct` | `REAL` | - | Annualized standard deviation of returns (Volatility) | `std_dev_ann_pct` |
| `max_drawdown_pct`| `REAL` | - | Maximum loss from a peak to a trough of a portfolio | `max_drawdown_pct` |
| `aum_crore` | `REAL` | - | Fund assets under management in crore INR | `aum_crore` |
| `expense_ratio_pct`| `REAL` | - | Operational expense ratio percentage | `expense_ratio_pct` |
| `morningstar_rating`|`INTEGER` | - | Morningstar star rating (1 to 5 stars) | `morningstar_rating` |
| `anomaly_flag` | `INTEGER` | - | Flag indicating abnormal returns (> 100% or null) | Cleaned derived |
| `expense_flag` | `INTEGER` | - | Flag indicating expense ratio outside range 0.1% - 2.5% | Cleaned derived |

### `fact_aum`
Historical Asset Under Management (AUM) by fund house. Populated from `data/raw/03_aum_by_fund_house.csv`.

| Column Name | SQLite Data Type | Constraint | Business Definition | Source Field |
| :--- | :--- | :--- | :--- | :--- |
| `aum_id` | `INTEGER` | Primary Key, Auto-increment | Unique identifier generated by the database | Generated |
| `fund_house` | `TEXT` | - | Asset Management Company name | `fund_house` |
| `date_id` | `INTEGER` | Foreign Key (`dim_date.date_id`) | Mapped date identifier | Mapped from `date` |
| `aum_crore` | `REAL` | - | AMC total AUM in crore INR | `aum_crore` |
| `num_schemes` | `INTEGER` | - | Number of schemes managed by the AMC | `num_schemes` |
